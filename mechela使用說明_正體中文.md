# Mechela 使用說明（繁體中文）

> 給不熟悉技術的使用者

---

## Mechela 是什麼？

Mechela 是一個跑在你自己電腦上的小工具，專門幫 NGO 的計畫管理或監測評估人員整理敘事報告裡的「改變證據」。

**它能幫你做的事：**
你把一份 Word 報告（.docx）丟進去，它會自動把裡面描述「事情真的改變了」的句子挑出來，讓你分類、整理成有來源、有時間軸的證據清單，最後可以直接匯出引用。

**它不會做的事：**
它不會替你寫報告、不會幫你下判斷，也不會把你的文件傳到任何地方。所有資料都只存在你自己的電腦裡。

---

## 它在解決什麼問題？

每到季末或年末要交報告，你可能有這種感覺：

- 「我知道這個計畫有進展，但我要花好幾個小時翻舊報告才能找到那段文字」
- 「這個改變是什麼時候開始的？從哪份報告開始出現的？」
- 「捐款方要我提供具體的改變證據，但我手邊只有一大堆敘述性的文字」

Mechela 的目標是把這個「從一堆文字裡挖出改變證據」的工作，從幾個小時縮短到幾分鐘。

---

## 使用前需要準備什麼

Mechela 需要你的電腦先裝好三個免費程式。以下是一步步的說明：

### 1. 安裝 Python

Python 是讓 Mechela 後端運作的程式語言。

1. 打開瀏覽器，前往 [python.org/downloads](https://www.python.org/downloads/)
2. 點選最上方的下載按鈕（版本 3.11 以上皆可）
3. 執行安裝程式，**安裝時務必勾選「Add Python to PATH」**（這個選項通常在安裝視窗下方，非常重要，沒勾會造成後面的步驟失敗）
4. 安裝完成

**確認方式：** 打開「命令提示字元」（Windows 鍵 → 搜尋「cmd」→ 開啟），輸入以下文字後按 Enter：
```
python --version
```
如果顯示 `Python 3.xx.x` 就代表成功了。

---

### 2. 安裝 Node.js

Node.js 是讓 Mechela 畫面（前端）運作的程式。

1. 前往 [nodejs.org](https://nodejs.org/)
2. 點選左邊標示「LTS」的版本下載（比較穩定）
3. 執行安裝程式，一路點「Next」即可

**確認方式：** 在命令提示字元輸入：
```
node --version
```
顯示版本號碼就代表成功。

---

### 3. 安裝 Git

Git 是用來把 Mechela 的程式碼下載到你電腦的工具。

1. 前往 [git-scm.com/downloads](https://git-scm.com/downloads)
2. 點選 Windows 版本下載並安裝（一路點「Next」即可，不需要更改任何選項）

**確認方式：** 在命令提示字元輸入：
```
git --version
```
顯示版本號碼就代表成功。

---

## 下載並安裝 Mechela

以上三個程式都裝好之後，進行以下步驟（只需要做一次）：

### 第一步：下載程式碼

打開命令提示字元，輸入以下兩行（每行輸入完按 Enter）：

```
git clone https://github.com/wcl-dev/mechela.git
cd mechela
```

這會在你目前的位置建立一個叫做 `mechela` 的資料夾，裡面是所有程式碼。

---

### 第二步：安裝套件

依序執行以下指令：

**後端套件：**
```
cd backend
py -m pip install -e ".[dev]"
```

**前端套件：**
```
cd ..\frontend
npm install
```

這步驟可能需要幾分鐘，讓它跑完就好。

---

### 第三步：啟動 Mechela

回到 `mechela` 資料夾，**直接雙擊 `start.bat`** 這個檔案。

它會自動開啟兩個黑色視窗（讓它們在背景跑，不要關掉），然後幫你開啟瀏覽器，進入 Mechela 的主畫面。

> 如果瀏覽器沒有自動開啟，請手動在網址列輸入 `http://localhost:3000`

**下次使用時，一樣雙擊 `start.bat` 就可以了。**

---

## 開始使用

### 建立專案

進入主畫面後，點選「**＋ New Project**」，輸入專案名稱（例如「東南亞人權計畫 2024」）。一個專案對應你的一個計畫或方案。

---

### 設定目標（Objective）

進入專案後，點選「**＋ Add Objective**」，輸入這個計畫正在追蹤的改變目標。例如：

> 「提升地方政府將人權納入政策的能力」

一個專案可以有多個目標。

---

### 上傳報告

點選「**＋ Upload Report**」，選擇你要分析的 Word 檔案（.docx 格式），填入報告名稱與日期，然後送出。

Mechela 會自動讀取整份文件，把可能代表「改變」的句子挑選出來。

---

### 審閱訊號（ABC Review）

上傳完成後，在報告旁邊點選「**Review**」。

左邊是系統挑出的「改變候選句」清單，點選任何一筆會在右邊展開詳細審閱面板。對每一筆，你需要做三件事：

**A — 確認信號文字**

信號文字是系統從報告裡擷取的原文。如果文字不夠精確，點「**Edit**」可以直接修改（例如刪掉多餘的前後文、改寫成更清楚的描述），改完按「**Save**」。

**B — 分類**

1. **調整改變的等級**（系統會預先判斷，你可以修改）：
   - **L1（已確立）**：改變已正式落地，例如「市政府通過了相關條例」
   - **L2（承諾中）**：有明確承諾或試行，例如「三個單位同意在下季納入新機制」
   - **L3（意識萌發）**：只有表達興趣或意識，例如「與會者對此框架展現興趣」
2. **決定這筆資料的狀態：**
   - 點「**Confirm**」— 確認這是一個真正的改變信號
   - 點「**Reject**」— 這不是改變（例如只是描述活動）
   - 點「**Context Signal**」— 這是外部環境的變化（如法規修訂），不算你的計畫成果

**C — 分配到改變線索（Thread）**

把信號分配到一個 Thread，形成同一主題的時間軸。系統會建議最相關的 Thread，你也可以點「**+ New thread**」建立新的。

---

### 重新分析（Re-analyse）

如果你在上傳報告之後才切換分析模式（例如從 Basic 升級到 Local AI），可以在專案頁面的 Reports 區塊，點選報告旁邊的「**Re-analyse**」按鈕，系統會用當前的分析模式重新偵測信號。

注意：重新分析會取代之前偵測到的所有信號（包括已確認的），你需要重新做一次 ABC Review。

---

### 什麼是改變線索（Thread）？

Thread 是你自己命名的主題分類，用來把不同報告裡描述同一件事改變的句子串起來。

舉例：你可以建立一個叫「地方政府政策倡議」的 Thread，然後把 Q1、Q2、Q3 報告裡所有跟這個主題相關的改變句都放進去。這樣你就有了一條有時間標記、有原文引用的改變軌跡。

---

### 撰寫進展摘要

在專案儀表板頁面，每個 Thread 下方會顯示「**+ Add progression summary**」的灰色文字。點擊後會展開一個文字編輯框，讓你用自己的話總結這條改變線索的整體進展。例如：

> 「從 Q1 立法委員表達興趣，到 Q2 提出修法草案，再到 Q3 通過委員會審查 — 數位治理法案正在穩步推進。」

寫好後按「**Save**」。已有摘要的 Thread 會以斜體顯示，點擊文字即可再次編輯。

這段摘要會出現在匯出的 Markdown 裡，放在 Thread 名稱和信號列表之間，成為你報告中可以直接引用的段落。

---

### 專案儀表板

儀表板是你的專案總覽頁面。上方有三個數字：Objectives 數量、Threads 數量、已確認信號數量。

下方依 Objective 分區顯示所有 Thread，每個 Thread 下方列出已分配的信號。這裡你可以：

- 閱覽整個專案的改變證據結構
- 為 Thread 撰寫或編輯進展摘要
- 確認哪些 Thread 有足夠證據、哪些還需要補充

---

### 匯出

在專案儀表板點選「**Export MD**」。頁面下方會展開一個「Markdown Export」區塊，內容包含：

- 每個 Objective 和它底下的 Thread
- 每個 Thread 的進展摘要（如果你有寫的話）
- 每個信號的文字、等級、出處報告和段落編號

點選「**Copy**」按鈕複製全文，貼到你的 Word、Google Docs、或 email 裡就是一份有來源的改變證據報告。

## 分析模式

Mechela 有三種運作模式，你可以在 Settings 頁面切換：

| 模式 | 需要什麼 | 效果 | 資料隱私 |
|------|----------|------|----------|
| **Basic（規則模式）** | 什麼都不需要 | 依照關鍵字比對，需要較多人工審閱 | 所有資料留在你的電腦 |
| **Local AI（地端 AI）** | 安裝 Ollama（免費） | AI 分析，準確度高 | 所有資料留在你的電腦，完全不上網 |
| **OpenAI（雲端）** | OpenAI API 金鑰（依用量計費） | 最高準確度 | 報告段落文字會傳送至 OpenAI 伺服器 |

### 設定 Local AI（推薦）

如果你希望用 AI 分析但不想把報告內容傳到外部伺服器，使用 Local AI 模式：

1. 在 `mechela` 資料夾裡，雙擊 `setup_local_ai.bat`（Windows）或在終端機執行 `./setup_local_ai.sh`（Mac/Linux）
2. 腳本會自動安裝 Ollama 並下載所需的 AI 模型（約 2.8 GB，只需下載一次）
3. 安裝完成後，到 Mechela 的 Settings 頁面，選擇「**Local AI**」，然後按「**Save**」
4. 按「**Test Connection**」確認顯示綠色的「Connected」

之後每次開機，Ollama 會自動在背景執行，你不需要再做任何設定。

### 設定 OpenAI（雲端）

如果你有 OpenAI 帳號和 API 金鑰：

1. 到 Settings 頁面，選擇「**OpenAI (cloud)**」
2. 在 API Key 欄位貼上你的金鑰（以 `sk-` 開頭），按「**Save**」

> 你的金鑰只存在你自己電腦的 `backend/user_settings.json` 裡。

### 不想設定？

直接使用 Basic 模式也完全可以運作，只是需要你多花一點時間在人工審閱上。

## 其他設定

### 自訂信號關鍵字（Custom Signal Keywords）

在 Settings 頁面的「Custom Signal Keywords」區塊，你可以加入你的領域常用的關鍵字，讓 Basic 模式（關鍵字比對）更準確。關鍵字按 L1 / L2 / L3 分類。

例如你的計畫常用「制度化」「試辦」「萌芽」等詞彙，加進去後系統就能在規則模式下自動辨認。

### 組織名稱（Organisation Names）

在 Settings 頁面的「Organisation Names」區塊，加入你的組織名稱和縮寫。這幫助系統區分「我們組織做了什麼」（活動）和「外部發生了什麼改變」（信號）。

例如加入「GFA」「Greenfield Alliance」，系統就會知道包含這些名稱的句子比較可能是在描述活動，而非外部改變。

## 常見問題

**Q：我的 Word 檔上傳失敗怎麼辦？**
請確認檔案是 `.docx` 格式（不是舊版的 `.doc` 或 PDF）。如果是從舊格式另存新檔，請確認儲存格式選擇的是「Word 文件（.docx）」。

**Q：啟動後瀏覽器一片白？**
請確認兩個黑色視窗都還開著，並且其中一個有顯示 `Application startup complete` 這段文字。如果沒有，等它跑完再重新整理瀏覽器。

**Q：出現「Port 8000 already in use」的錯誤？**
代表另一個程式正在佔用同一個通訊埠。請打開工作管理員，找到佔用的程式並結束它，然後重新啟動 Mechela。

**Q：我的資料會被上傳到網路嗎？**
看你選擇的模式。使用 Basic 模式或 Local AI 模式時，所有資料完全留在你的電腦裡，不會傳到任何地方。使用 OpenAI 模式時，報告的段落文字會送到 OpenAI 進行分析。如果你的報告含有高度敏感的保密資訊，請使用 Local AI 或 Basic 模式。

**Q：可以同時分析多個計畫的報告嗎？**
可以。每個計畫建立一個「Project」，各自獨立管理。
